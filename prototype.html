Idea Development

Our system grows from studying natural feline behaviors and translating them into robotic functions that support a cat’s physical and emotional needs.

Key Points

  • Inspired by wild cat instincts: play, hunt, smell, clean
  • Each instinct maps directly to a corresponding robot module
  • Cat-owner pain points guide functional priorities
  • Base station acts as the “territory center” for resources and coordination

Technical Diagrams

This part demonstrates the technical details of a modular swarm-robotics ecosystem. The system architecture diagrams and section views below illustrate how hardware and mechanisms integrate to support autonomous, coordinated, and safe interactions with cats.

1. Sensing Layer

Purpose

Provides environmental awareness and ensures safe navigation when robots interact with cats, humans, and furniture.

Description

This diagram shows how infrared, ultrasonic, and ToF sensors work together to provide cross-validated distance perception, enabling the robots to detect obstacles accurately at different ranges and under different lighting conditions. BLE/RFID modules allow robots to recognize nearby units and maintain spacing, while the IMU helps stabilize the robot by detecting tipping or unexpected impacts.

2. Mesh Communication Layer

Purpose

Allows all robots to share status, synchronize actions, and operate as a coordinated swarm.

Description

This diagram illustrates the hybrid communication system combining Bluetooth Mesh, ESP-NOW, and BLE Advertising. Each protocol supports a different communication need: global state sharing, ultrafast action-level updates, and lightweight broadcasts. Together, they ensure every robot receives consistent information and can respond in real time to environmental changes or task transitions.

3. Functional Diagram & Section View

3.1 Playing Robot

Purpose

Shows how sensing, actuation, and control modules integrate to create an engaging, fast-moving play behavior for the cat.

Description

This diagram highlights how the Playing Robot combines vertical lift actuation, high-torque motorized movement, position sensing, and optional LED/audio modules to create dynamic chase interactions. The section view on the right reveals the mechanical stacking of sensors, motors, encoders, and the ESP32 controller, illustrating how the entire system supports precise movement and safe play.

3.2 Cleaning Robot

Purpose

Demonstrates how the robot performs floor cleaning tasks by integrating brushes, suction modules, and movement control.

Description

This diagram shows how the Cleaning Robot uses a roller brush, edge brushes, and a mini vacuum module to collect fur and debris. Wheel encoders and sensing data guide the robot’s path to prevent repeated cleaning of the same area. The section view reveals the layout of the vacuum chamber, motors, sensors, and ESP32 electronics that support coordinated cleaning behaviors.

3.3 Scent Robot

Purpose

Explains how the robot produces and distributes calming scents during interactions with the cat.

Description

This diagram presents the scent-dispersion system composed of an ultrasonic aroma diffuser, a Peltier heating module for boosting scent intensity, and a low-noise airflow fan for smooth diffusion. The section view demonstrates how these components integrate with the sensing and motor layers, allowing the robot to position itself beside the cat before releasing the scent.

3.4 Feeding Robot

Purpose

Shows how the robot safely dispenses dry food in controlled amounts.

Description

This diagram illustrates the food-dispensing mechanism built around a stepper motor, a servo-controlled lid, and a load cell for measuring remaining food. Ultrasonic and ToF sensors detect the cat’s presence to trigger dispensing. The structural section view highlights the physical arrangement of the food storage container, dispensing unit, electronics, and movement system.

4. Base Station Layer

Purpose

Supports all robots with wireless charging, robot identification, and long-range connectivity.

Description

This diagram shows the architecture of the base station, which integrates Qi wireless charging coils, NFC docking identification, a power-management board, and a LoRa/BLE gateway for extended communication range. The section view displays how these elements stack to form a unified hub that charges the robots, identifies which unit has docked, and relays network data.

Product Design

Design Sketch Exploration

Our early sketches explore different designs, modular arrangements, and transformation possibilities. These drawings helped us define how each robot might move, combine, and shift form while maintaining a cohesive system identity.

  • Iterated multiple silhouettes to find the right softness
  • Explored modular clustering and separation
  • Used fast sketching to discover overall form direction

Bubble-Inspired Form Language

The naturally merging shapes of soap bubbles inspired our soft and playful form language. Their seamless clustering informed how our robot modules should blend visually when grouped.

  • Soft, organic curvature for a friendly aesthetic
  • Bubble clusters guide how modules visually merge
  • Avoids mechanical appearance, emphasizes playfulness
  • Supports visual unity when robots reorganize dynamically

3D Form Development

We created 3D models to refine the soft, playful bubble-like geometry. The forms maintain a cohesive aesthetic whether the modules stand alone or cluster together as a group.

Morphological Arrangement

We simulated multiple arrangement patterns to study how the four modules re-organize during movement. These patterns ensure the formations remain intentional and visually coherent even as robots transition between tasks.

  • Explored linear, triangular, and clustered configurations
  • Verified smooth transitions between arrangements during motion

Material Strategy

Material selection balances softness, safety, and internal structural strength. This ensures the robots feel approachable while remaining durable and mechanically reliable.

  • Fabric-Reinforced Silicone: soft, gentle, furniture-safe, cat-safe, playful texture
  • ABS Structure: rigid internal support, durable, required for electronic protection

Challenges and Aspirations

Technical Part

Challenges

Maintaining Stable Multi-Robot Communication

It might be difficult to keep all four robots synchronized when they moved quickly or were separated by furniture, as wireless signals weakened or arrived at different times.

Sensor Reliability Around Real Cats

Fur and sudden motion might caused IR, ultrasonic, and ToF sensors to produce inconsistent readings, making obstacle detection less reliable.

Fitting Hardware into a Small Soft Form

Motors, sensors, batteries, and PCBs all needed to fit inside a small rounded shell, probably creating heat, spacing, and mounting challenges.

Aspirations

Smarter Behavior Adaptation

We hope the robots can one day adjust actions based on the cat’s activity patterns or mood signals, rather than following fixed routines.

More Autonomous Swarm Coordination

We aim to develop self-organizing swarm behaviors so robots can switch roles or recover from errors without manual control.

More Robust Fault Detection & Recovery

We aim to allow each robot to detect motor stalls, sensor failures, or communication drops, and recover actions automatically without interrupting the swarm.

Product Design Part

Challenges

Balancing Soft Aesthetics with Functional Openings

Adding vents, sensors, or feeding openings risked breaking the smooth bubble-like form we wanted to maintain.

Consistent Aesthetics Across Different Formations

Because robots constantly move and regroup, some arrangements looked unbalanced until proportions and spacing were refined.

Material Balance Between Softness and Structure

Silicone offered softness but lacked precision; ABS offered structure but risked looking too rigid.

Aspirations

More Expressive and “Alive” Surfaces

We hope future versions can include subtle movements or lighting cues to give the robots a more lifelike presence.

Clearer Visual Identity When Robots Cluster

We aim to use color gradients or geometric cues that create recognizable patterns when modules form groups.

More Durable Soft Materials for Real Pet Use

We want materials that stay soft but are more resistant to scratches, bites, and long-term deformation.
